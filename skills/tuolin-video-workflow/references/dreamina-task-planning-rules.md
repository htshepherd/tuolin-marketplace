# 即梦任务规划规则

本文件用于生成 `dreamina_generation/dreamina_jobs.md/json` 前的任务验证。

## 任务类型

- `image2video`：用真实产品图或应用图生成镜头。
- `text2video`：仅用于环境或过渡镜头，不得生成具体产品可见镜头。
- `reuse_video`：直接复用真实视频素材。
- `reuse_image`：复用静态图作为镜头素材。
- `blocked`：素材缺失或规则不允许生成。

## 验证要求

- 可见产品镜头不能是 `text2video`。
- 可见产品的 AI 生成镜头必须绑定真实产品参考素材。
- 镜头时长由已确认分镜动态决定，通常 4-8 秒；允许范围必须来自 `dreamina_capability_profile`，默认 4-15 秒。
- 即梦任务的模型、分辨率和画幅必须来自 `dreamina_capability_profile`，默认 `seedance2.0_vip`、`1080P`、`9:16`。
- `dreamina_capability_profile` 必须记录图片、视频、音频和总文件数上限；后续多模态素材编号引用和用途校验必须读取同一 profile。
- 每个任务必须继承 `prompt_quality_checks`。如果 Prompt 存在指令冲突、娱乐化模板、缺失分时段/产品展示模板，或镜头时长承载不了当前场景复杂度，任务状态必须 blocked。
- 每个任务必须记录预计额度。
- 付费提交前必须等待 `确认即梦生成`。
- 一个镜头可包含一张或多张有序参考图，每张图必须记录连续性角色和编号。
- 若当前 provider profile 未明确支持有序多图输入，任务必须 blocked，并要求把该镜头改写为连续相邻镜头后重新确认分镜和 SRT；不得静默丢图或自动拆分。
- 任务计划、额度和 blocker 必须直接打印在 Codex 对话中供确认，不能只返回文件路径。
- 真实提交完成后由用户在即梦网页端监测。用户说明全部生成后，系统只执行一次基于真实 task ID 的下载与完整性校验，不暴露轮询步骤。
- 本期不实现单镜头重做或结果改进；不接受结果时记录原因并停止当前运行。
