# PRD：视频知识档案与下游实时素材使用链路

**日期：** 2026-07-16
**状态：** 已完成领域决策，待开发
**上位文档：** `doc/PRD-2.0.md` draft-12
**首期产品范围：** 石英纤维隔热带

## 1. 背景

当前代码只完成了视频文件登记、可选的单点抽帧和基于缓存图片存在性的处理状态统计：

- 视频作为 `content_asset` 和 evidence 来源登记；
- 独立抽帧工具默认只取 `00:00:00`；
- 产品整理主流程不会自动理解视频内容；
- 抽帧图片没有正式知识卡、关键片段或 Agent 接口索引；
- 视频创作上下文过滤视频素材，只使用图片素材；
- 下游无法稳定回答某条视频讲了什么，也无法通过受控引用反向截取精确帧或片段。

因此，“素材已登记”不等于“视频已处理”，更不等于下游视频创作 Agent 可以使用。

## 2. 目标

建立以下完整链路：

```text
raw 源视频（保持不变）
  → 视频素材登记与稳定 Asset ID
  → 自适应分析帧/必要分析短片
  → Codex 视频语义复核
  → 持久 Video Content Profile
  → Agent 接口目录与详情
  → 下游按授权检索和选择
  → 从 raw 实时提取任务帧/任务片段
  → 分镜预览与真实视频组装
```

下游至少能够获知：

- 视频讲了什么；
- 哪些时间段和时间点存在关键动作；
- 产品在各片段是否清晰可见、是否足以确认身份；
- 视频属于产品展示、安装、应用、测试还是生产过程；
- 可支持哪些稳定镜头用途；
- 3–6 张代表帧；
- 画面、构图、人物、标识和音频风险；
- 可观察的测试过程、前后状态或变化；
- 哪些结论只是观察，哪些需要正式证据或人工复核。

## 3. 非目标

- 不移动、删除、改名、覆盖或重新编码 raw 源视频。
- 不让下游 Agent 浏览 raw 目录或提交本地 raw 路径。
- 不把视频观察自动升级为产品参数、认证、性能、安全或测试结论。
- 不为当前版本配置独立多模态 API Key。
- 不在首期部署持久视觉向量模型。
- 不一次性无边界处理全部 164 条视频。
- 不把任务抽取帧或片段自动写回正式知识。
- 不通过本项目删除重复 raw 视频。

## 4. 核心领域模型

### 4.1 Video Asset Card

沿用 `content_asset` 卡型，负责：

- raw 文件登记；
- 不透明 `Video Asset ID`；
- 产品关系；
- 固定顶层类别和完整子文件夹路径；
- 当前私有 raw 路径映射；
- 授权、撤销和来源修订；
- 与 `Video Content Profile` 的一对一关系。

Asset ID 不从路径、文件名或内容哈希生成。确认是移动时保留 ID；并存副本获得新 ID并进入 `Video Asset Family`；同路径内容替换保留 Asset ID，但创建新来源修订并使旧档案失效。

### 4.2 Video Profile Card

新增正式卡型 `video_profile`，负责一条视频的持久语义档案。

正式文件：

```text
knowledge/okf/视频档案/{product_slug}/{video_asset_id}.md
knowledge/okf/视频档案/{product_slug}/{video_asset_id}.json
```

两份文件必须由同一个已校验领域对象原子生成：

- Markdown：供员工阅读、抽查和复核；
- JSON：保存完整时间结构、风险、代表帧引用、修订和处理来源，供校验与 Agent 接口生成。

缺少任一文件、修订不一致、内容摘要不一致或关键字段冲突时，档案无效。

### 4.3 Video Content Profile 最低字段

- `profile_id`
- `video_asset_id`
- `product_id`
- `profile_revision`
- `source_revision`
- `source_classification`
- `observed_classifications`
- `title`
- `summary`
- `duration`
- `media_facts`
- `key_segments`
- `anchor_moments`
- `representative_frames`
- `use_capabilities`
- `audio_observations`
- `transcript_detail`
- `source_audio_use_policy`
- `observation_confidence`
- `risk_summary`
- `evidence_links`
- `processing_state`
- `analysis_completeness`
- `analysis_provenance`
- `exclusions`

### 4.4 Observation 与 Claim 分离

`Video Observation` 只描述直接可观察内容，例如：

- 人员将隔热带缠绕在管道外；
- 火焰靠近样品；
- 某时间段出现前后颜色变化；
- 产品在画面中清晰、部分遮挡或无法确认身份。

涉及产品身份、测试成功、耐温、安全、认证、因果关系或性能的内容属于 `Video Claim Interpretation`，没有正式证据或人工复核时不能成为对外事实。

## 5. 来源文件夹分类

固定顶层文件夹和完整子文件夹路径必须进入 `Video Source Classification`。

首期固定处理范围：

1. 产品视频；
2. 应用场景素材；
3. 测试验证素材；
4. 其他已确认属于石英纤维隔热带的固定素材文件夹。

应用场景目录规则：

- “应用场景素材”下的第一级子文件夹是标准 `Source Application Scenario`；
- 更深层子文件夹按顺序保留为来源上下文；
- 文件夹名称可提高分析先验和检索相关性，但不能替代画面观察；
- 来源分类与视觉观察冲突时必须同时保留并给出复核状态。

## 6. 视频理解流水线

### 6.1 批次能力预检

批次启动前必须验证：

- `ffprobe` 可读取媒体事实；
- `ffmpeg` 可抽帧、生成分析短片和运行时截取；
- 当前 Codex 会话可以查看准备好的视觉输入；
- 配置和目录允许写入 generated 暂存与缓存；
- 可选 ASR、质量检测和去重能力状态。

缺少任一必需能力时，批次进入 `blocked`，不消耗批次处理，不把视频标记为失败。

### 6.2 确定性媒体检查

`ffprobe` 提供：

- 时长；
- 分辨率；
- 帧率；
- 编码；
- 旋转；
- 视频流和音频流；
- 可读性错误。

模型不得猜测这些媒体事实。

### 6.3 两阶段自适应抽帧

第一阶段：全片均匀覆盖，建立开头、发展、结尾和主要章节。

第二阶段：围绕以下变化增加候选：

- 场景切换；
- 主体变化；
- 产品进入或离开画面；
- 产品可见度变化；
- 动作开始、高潮、结束；
- 安装、测试、生产的过程状态变化；
- 前后状态变化；
- 可能直接复用的连续动作。

处理要求：

- 替换或排除黑帧、严重模糊帧和近重复帧；
- 安装、测试、生产视频尽量覆盖前、过程、后；
- 默认软预算为最多 24 张分析帧；
- 复杂或长视频按章节分段或明确升级深度；
- 不得因为达到预算而静默丢失主要内容。

### 6.4 分析短片

静态产品展示、慢速场景和只适合取帧的视频默认只使用有序分析帧。

以下情况才生成 3–12 秒 `Video Analysis Clip`：

- 连续动作无法由静态帧可靠判断；
- 动作边界不清；
- 前后变化是否持续需要验证；
- 快速运动导致帧间解释冲突；
- 可能直接复用为真实镜头。

每条视频默认最多 6 段分析短片。

### 6.5 Codex 视频语义复核

当前正式语义分析执行者是 Codex：

- 不要求用户配置独立模型 API Key；
- 不允许脚本暗中调用其他云端多模态服务；
- Codex 接收预算内帧、必要短片、媒体事实、文件夹分类和可用转录；
- Codex 不获得通用 raw 目录浏览权限；
- Codex 输出必须转为固定 profile schema；
- 原始模型输出不能直接发布，必须通过来源、时间范围、字段、风险和修订校验。

### 6.6 音频与 ASR

语音识别使用可替换、本地优先的 `Speech Transcription Adapter`。

输出至少包括：

- 语言；
- 带时间戳文本；
- 置信度；
- 可用时的说话人分段；
- 未识别区间；
- 工具和模型版本。

无语音时可跳过转录，但保留环境声、机器声、噪声和音乐观察。

有清晰语音但 ASR 不可用时：

- 视觉档案可在 `Visual Observation Scope` 下完成；
- 必须标记音频理解不完整；
- 不得声称没有重要语音；
- 原音频默认 `human-review-required` 或 `mute-recommended`。

## 7. 关键片段、锚点与代表帧

### 7.1 Video Key Segment

每个关键片段必须记录：

- 起止时间；
- 可见主体；
- 动作或过程阶段；
- 画面描述；
- 产品可见度；
- 构图与裁剪安全；
- 人物和标识风险；
- 音频状态；
- 是否要求动作连续；
- 可复用模式：片段、帧、两者或不可用；
- 编辑适用性：ready、adaptable、reference-only、unusable。

### 7.2 Video Anchor Moment

关键片段内记录一个或多个精确时间点，用于：

- 代表帧；
- 用户复核；
- 运行时小范围探索；
- 精确任务帧截取。

### 7.3 Video Representative Frame

每条正式档案提供 3–6 张代表帧：

- 优先来自首选关键片段锚点；
- 覆盖不同主要内容和用途；
- 安装/测试尽量覆盖前、过程、后；
- 可少量包含全局环境或主体建立帧；
- 不因画面漂亮而全部选自同一重复动作。

代表帧存放在可重建 generated 存储中，通过修订绑定的受控引用提供，不暴露缓存目录。

## 8. Agent 接口

### 8.1 三级读取

1. `Video Profile Catalog`
   - 标题和短摘要；
   - 来源/观察分类；
   - 使用能力；
   - 可见度；
   - 复用模式；
   - 风险摘要；
   - 档案状态；
   - 轻量代表帧缩略图引用。

2. 完整档案详情
   - 全部关键片段；
   - 锚点；
   - 3–6 张代表帧引用；
   - 风险、音频、转录、证据和处理来源；
   - 结构化机器数据。

3. 运行时素材提取
   - 通过运行 ID、接口修订、Asset ID、profile 修订、操作和时间范围授权；
   - 读取 raw 但不修改 raw；
   - 输出只写入当前视频创作运行目录；
   - 每次尝试记录审计，包括拒绝请求。

### 8.2 代表帧图片交付

- 目录层不内嵌 Base64；
- 详情层返回受控图片引用、尺寸、内容指纹、时间点和所属片段；
- 档案失效、排除或撤销后引用不可继续使用；
- 本地阶段可用受控文件引用，未来可替换为授权媒体端点而不改变下游契约。

### 8.3 检索

当前实现：

```text
产品/授权/状态/风险硬过滤
  → 结构化与文本召回
  → 加载小规模候选档案
  → Codex 查看代表帧进行视觉复排
```

接口明确提供：

- `visual_vector_index_unavailable`
- `codex_visual_rerank_available`

首期不部署视觉向量模型。未来增加向量索引时只能改善初步召回，不能改变档案观察或人工结论。

## 9. 下游视频创作使用

视频创作 Agent 保持 `raw_access=false`。

它可以：

- 搜索已授权的有效视频档案；
- 查看摘要、关键片段和代表帧；
- 将视频使用能力匹配到当前策划或镜头；
- 从已授权关键片段提取最多 3 个低成本候选预览；
- 选中后生成精确任务帧或任务片段；
- 在分镜确认前显示真实任务预览；
- 将适合的真实片段直接用于组装。

它不可以：

- 提交任意本地视频路径；
- 重新扫描整条视频；
- 绕过关键片段范围无限抽帧；
- 将代表帧冒充最终任务帧；
- 把视频观察当作产品事实；
- 在档案撤销后继续提交或使用对应素材。

运行时探索从锚点开始，只在授权关键片段内最多提取 5 张候选帧。最终选中帧必须重新检查主体、清晰度、构图、裁剪、重复、标识和人物风险。

## 10. 真实片段编辑边界

允许自动执行的意义保持型处理：

- 精确裁切时间范围；
- 按旋转元数据纠正方向；
- 高质量转码；
- 分辨率和帧率规范化；
- 已确认的裁剪或留边；
- 按音频策略静音。

以下操作可能改变动作感知或真实性，必须披露并在适用时确认：

- 加速或减速；
- 稳定；
- 调色、锐化、降噪；
- 删除片段内部动作；
- 镜像；
- 重排；
- 补帧；
- 生成式延长或移除来源标识。

不得改变测试状态、前后顺序、产品身份或观察含义。

## 11. 风险与测试素材

每个关键片段记录：

- 产品可见度；
- 构图安全；
- 字幕、水印、平台 UI、第三方、客户、项目和个人标识；
- 人脸、未成年人和授权状态；
- 原音频使用策略；
- 编辑适用性。

测试验证视频最后处理。

对外使用测试片段时：

- 结论必须关联正式证据，并受证据范围约束；
- 无正式证据时只能使用经复核的中性可见描述；
- 产品身份、条件或前后关系不清时阻止外部使用；
- 不得通过剪辑隐藏不利结果或制造更有利的前后对比；
- 无证据的口头测试结论必须静音或排除。

## 12. 批次、检查点与发布

### 12.1 处理顺序

1. 选择一条真实低风险产品视频完成端到端 tracer；
2. 处理并全量验收 8 条产品视频；
3. 应用场景按第一级场景子文件夹逐批处理；
4. 最后处理测试验证视频。

### 12.2 单视频原子检查点

用户一次确认一个业务批次。系统内部逐条视频处理，并在每条完成后保存：

- 准备输入清单；
- Codex 语义建议；
- Markdown 档案草稿；
- JSON 结构；
- 代表帧引用；
- 校验结果；
- 处理状态。

检查点路径：

```text
generated/staging/video-profiles/{batch_id}/{product_slug}/{video_asset_id}.md
generated/staging/video-profiles/{batch_id}/{product_slug}/{video_asset_id}.json
```

中断恢复时，来源指纹和策略版本未变化的有效检查点不重复分析。

### 12.3 批次验收

- 产品视频首批 8 条全部复核；
- 每个应用场景首批不超过 10 条时全量复核，超过 10 条至少复核 5 条；
- 稳定后的增量批次抽查 10%，最少 3 条；
- 测试素材外部风险字段全部复核；
- 系统性抽帧、分类、时间、音频或风险错误阻止批次验收。

### 12.4 事务式发布

暂存检查点不进入下游接口。

验收通过后：

1. 原子提升允许发布的 Markdown/JSON 档案；
2. 重建 Agent 接口；
3. 校验目录、详情、代表帧和修订；
4. 校验成功后激活新接口版本。

任何一步失败：

- 本批次状态为未完成；
- 不激活新接口；
- 下游继续使用上一已验证版本。

## 13. 状态与进度

单视频处理状态：

- `registered`
- `processing`
- `valid`
- `review_required`
- `failed`
- `excluded`
- `stale`

产品进度分别报告：

- 已登记视频；
- 有效档案；
- 待处理；
- 失败；
- 待复核；
- 已排除。

旧 `generated/cache/video-frames/` 单点缓存不再计为已处理。只有当前来源、分类、schema 和分析策略匹配的有效档案才计入完成。

## 14. 缓存留存

- 当前有效档案引用的 3–6 张代表帧随该档案修订保留；
- 其他分析帧和短片至少保留到批次验收完成，随后保留 30 天；
- 待复核、失败、争议或调查中的缓存问题解决前不清理；
- 任务帧、任务片段和任务预览跟随视频创作运行目录；
- 清理必须依据缓存 manifest、来源指纹和引用关系；
- 清理永远不能作用于 raw。

## 15. 首期端到端验收

必须使用一条真实且未修改的产品视频证明：

1. raw 哈希在流程前后不变；
2. 建立 Asset ID 和来源修订；
3. 生成自适应分析素材；
4. Codex 生成并校验视频档案；
5. 生成 3–6 张代表帧；
6. 事务式发布并验证 Agent 接口；
7. 视频创作 Agent 通过目录和详情检索到该视频；
8. 通过 Asset ID 提取精确任务帧和片段；
9. 分镜显示实际任务预览；
10. 真实片段进入组装输入；
11. 任意 raw 路径请求被拒绝；
12. stale、excluded 或 revoked 档案不能继续提取或进入提交。

只完成 schema、脚本、缓存或 mock 不算交付。

## 16. 成功标准

- 下游无需扫描 raw 即可判断视频的主要内容和具体可用片段。
- 每条有效视频档案至少提供标题、摘要、关键片段、锚点、3–6 张代表帧、风险和使用能力。
- 视频创作 Agent 能从真实 raw 视频生成任务帧或片段，但无法获得 raw 路径。
- 视频观察与产品事实边界在档案、检索、分镜和外部使用阶段均被执行。
- 知识写入后必须完成 Agent 接口刷新和验证；失败时不会暴露部分新状态。
- raw 文件在分析、发布、运行时提取和缓存清理后保持不变。
